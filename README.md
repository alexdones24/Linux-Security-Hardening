A Linux security hardening project completed using **Rocky Linux 9** in **Oracle VirtualBox**.

This repository contains the reports and assignments I completed while learning Linux security hardening. Throughout the project, I worked with different security tools and frameworks to assess, secure, and improve a Rocky Linux virtual machine.

---
## Overview
Over the course of this project, I learned how to build and secure a Linux virtual machine using industry-recognized tools and security guidelines. Starting with setting up a Rocky Linux 9 virtual machine, I performed security assessments, manually remediated security findings, explored automated hardening with Ansible, and worked with SELinux.

The reports in this repository document each stage of the project and include screenshots of my work and the results I achieved.

## Project Breakdown

### Virtual Machine Setup
- Created a Rocky Linux 9 virtual machine in Oracle VirtualBox
- Evaluated Linux distributions before selecting Rocky Linux
- Configured the virtual machine for future security labs

### Security Assessment
- Performed compliance scans using DISA STIG and CIS Benchmarks
- Reviewed security findings using the SCAP Compliance Checker

### Security Remediation
- Researched and fixed security findings
- Improved the system's compliance with security recommendations

### STIG Viewer
- Reviewed CAT I, CAT II, and CAT III findings
- Documented manual remediation steps

### Automated Hardening
- Used Ansible to automate the application of DISA STIG configurations
- Compared manual hardening with automated configuration management

### SELinux Administration
- Explored SELinux modes, labels, contexts, and policy booleans
- Learned how SELinux provides additional access control for Linux systems
